# Week 2 — 여러 명이 동시에: 브로드캐스트 단체 채팅

여러 명이 동시에 접속해서, 한 사람이 보낸 말을 **접속한 모두**가 받는 단체 채팅입니다.
이 주차도 일부러 **클래스 없이** 함수 + 전역 목록 + 스레드로만 작성했습니다.

## 파일
| 파일 | 설명 |
|------|------|
| `server.py` | 손님마다 스레드를 붙여 받고, 받은 말을 전원에게 뿌린다(broadcast) |
| `client.py` | 보내기(메인) + 받기(스레드)를 동시에 |
| `make_ppt.py` | 강의자료 PPT 생성 스크립트 (강사용) |
| `Week02_여러명이_동시에.pptx` | 완성된 강의 슬라이드 (발표자 노트 포함) |

## 실행 방법
서버 1개 + 클라이언트 **여러 개**(터미널을 3개쯤) 띄웁니다.

```bash
# 터미널 1 — 서버
python server.py

# 터미널 2, 3, 4 ... — 사람마다 하나씩
python client.py
```

각 클라이언트는 시작할 때 닉네임을 입력합니다. 한 명이 친 말이 나머지 모두의
화면에 동시에 뜹니다. 누가 들어오고 나가는지도 모두에게 알림이 갑니다. 종료는 `Ctrl+C`.

## 핵심 개념
- **스레드(threading)** — 손님마다 전담 직원. 동시에 여러 사람을 응대한다.
- **브로드캐스트** — 한 명에게 받은 말을 접속자 전원에게 다시 뿌리기.
- **서버 = 중앙 우체국** — 모든 말은 서버를 거쳐 모두에게 배달된다.
- **닉네임을 '첫 줄'로 약속** — 우리만의 규칙(프로토콜)의 시작.

## 실습 / 과제
1. 내 메시지가 화면에 한 번만 보이게 — 보낸 사람을 제외하고 브로드캐스트
2. 접속자 수를 알려주는 명령 (예: `/count`)
3. 누군가 나가면 남은 사람 목록 함께 보여주기

> 생각해보기: 메시지가 길어 `recv` 한 번에 다 안 오면 어떻게 될까요?
> (TCP 메시지 경계 문제 — 뒤 주차에서 프로토콜로 해결합니다.)
